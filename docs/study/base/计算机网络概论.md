# 计算机网络概论

## 协议

协议定义了两个或多个通信实体之间交换的报文**格式**和**顺序**，以及报文发送和/或接收一条报文或其他事件所采取的动作。

### HTTP 1.1

采用同时发送三个 TCP 连接。

缺点：

- 无法多路复用

### HTTP 2

帧，可以标识某个数据为某个请求，可以多路复用，同时可进行头部压缩。

缺点：

- 在 TCP 上如果丢包了，会出现队头堵塞。
- TCP 和 TLS 需要各自握手，耗费时间长。

### HTTP 3

QUIC （quick UDP internet connection）。将 TLS 作为自身的一部分，解决 TCP 和 TLS 需要各自握手的问题。同时拥有独立的流，解决了 TCP 队头堵塞的问题。

UDP 不靠谱但是 QUIC 靠谱。（UDP 不管丢包和顺序）

### websocket

websocket 是一种在单个 TCP 连接上进行全双工通信的协议。它实现了浏览器与服务器全双工通信，允许服务器主动向客户端推送信息。

## CDN

### DNS 劫持

将要加速的域名重定向到 CDN 厂商的域名解析服务处理，CDN 厂商根据来源确定最近的 CDN 服务器的 IP，用户直接访问最近的 CDN 服务器。
